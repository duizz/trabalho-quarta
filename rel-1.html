Relatório do Projeto: Construção de um Semáforo Básico com Arduino

Integrantes: João Pedro Costa, Filipe André Nunes, Julio Cesar Florencio, Luiz Felipe Guerra, Felipe Silva da Silva.

1. Objetivo do Projeto

O objetivo deste projeto é construir um semáforo básico utilizando a plataforma Arduino. A proposta inclui:

  • Testar o circuito no Tinkercad, uma ferramenta de simulação online para projetos de eletrônica.
  • Usar o Arduino Cloud para conectar o Arduino à máquina física, permitindo controlar o semáforo remotamente.

2. Ferramentas e Materiais Utilizados

  • Arduino UNO: A placa principal usada para controlar o semáforo.
  • LEDs: Três LEDs (vermelho, amarelo e verde) para simular as luzes do semáforo.
  • Resistores: Para limitar a corrente que passa pelos LEDs.
  • Protoboard e Jumpers: Para conectar os componentes eletrônicos.
  • Tinkercad: Plataforma online para simulação do circuito.
  • Arduino Cloud: Serviço de nuvem que permite a conexão remota e monitoramento do Arduino físico.

3. Montagem do Circuito

O circuito foi montado conforme a seguinte estrutura:

  • Os três LEDs (vermelho, amarelo e verde) foram conectados aos pinos digitais do Arduino (pinos 8, 9 e 10, respectivamente).
  • Cada LED foi conectado em série com um resistor de 220Ω para proteger os LEDs contra sobrecarga de corrente.
  • O protoboard foi usado para facilitar as conexões entre os componentes.
  • O circuito foi simulado e testado no Tinkercad para garantir que as ligações e o código estivessem corretos.

4. Código do Arduino

O código foi escrito na IDE do Arduino, utilizando funções básicas para alternar entre os estados do semáforo. O loop principal do código controla o tempo que cada LED permanece aceso, simulando o funcionamento real de um semáforo.

Exemplo de Código:

int red = 8;
int yellow = 9;
int green = 10;

void setup() {
  pinMode(red, OUTPUT);
  pinMode(yellow, OUTPUT);
  pinMode(green, OUTPUT);
}

void loop() {
  digitalWrite(green, HIGH);
  delay(5000); // 5 segundos
  digitalWrite(green, LOW);

  digitalWrite(yellow, HIGH);
  delay(2000); // 2 segundos
  digitalWrite(yellow, LOW);

  digitalWrite(red, HIGH);
  delay(5000); // 5 segundos
  digitalWrite(red, LOW);
}

5. Teste no Tinkercad

Após a montagem do circuito e do código, o projeto foi simulado no Tinkercad. Essa plataforma permite verificar o funcionamento do semáforo antes de conectar ao hardware físico. Durante o teste, foram feitas algumas verificações:

  • Temporização correta das luzes.
  • Transições suaves entre os estados do semáforo.

6. Conexão com o Arduino Cloud

A próxima etapa foi conectar o Arduino ao Arduino Cloud para possibilitar o controle remoto. A integração com a nuvem foi feita para monitorar o estado do semáforo e, potencialmente, enviar comandos para modificar o comportamento do semáforo em tempo real.

Passos:

  • Criar um dispositivo no Arduino Cloud e configurar a conexão com o Arduino.
  • Programar o código para sincronizar os estados do semáforo com a nuvem.
  • Testar a interação remota e garantir que as alterações no painel do Arduino Cloud refletissem no hardware físico.

7. Resultados e Conclusões

O projeto de construção de um semáforo básico usando Arduino foi bem-sucedido. A simulação no Tinkercad ajudou a validar o circuito e o código antes de implementá-los no hardware físico. A integração com o Arduino Cloud permitiu monitorar e controlar o semáforo remotamente, mostrando a flexibilidade e a conectividade da plataforma Arduino.
