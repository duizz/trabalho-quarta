Relatório do Projeto: Sensor de Proximidade com Arduino e LED

Integrantes: João Pedro Costa, Filipe André Nunes, Julio Cesar Florencio, Luiz Felipe Guerra, Felipe Silva da Silva.

1. Objetivo do Projeto

O objetivo deste projeto é desenvolver um sistema de detecção de proximidade utilizando um sensor ultrassônico HC-SR04 e um Arduino. O sistema deve acionar um LED sempre que um objeto estiver a uma distância de 10 cm ou menos do sensor e desligar o LED quando o objeto estiver a uma distância maior. Este projeto simula um detector de obstáculos simples, com uso prático em robótica e sistemas de automação.

2. Ferramentas e Materiais Utilizados

  • Hardware:
    • Arduino Uno R3
    • Sensor Ultrassônico HC-SR04
    • LED
    • Resistor de 220Ω
    • Jumpers (fios de conexão)
    • Protoboard (opcional, para facilitar a montagem)
  • Software:
    • Ambiente de simulação Tinkercad, que permite criar e testar circuitos eletrônicos de forma virtual.
    • IDE do Arduino (no Tinkercad), para escrever e simular o código de controle do sistema.

3. Montagem do Circuito

O circuito foi montado conforme descrito abaixo: Conexões do Sensor Ultrassônico HC-SR04:

  1. VCC (pino do sensor) conectado ao 5V do Arduino.
  2. GND (pino do sensor) conectado ao GND do Arduino.
  3. Trig (pino do sensor) conectado ao pino digital 9 do Arduino.
  4. Echo (pino do sensor) conectado ao pino digital 10 do Arduino.

Conexões do LED:

  1. Anodo (perna longa do LED) conectado ao pino digital 13 do Arduino.
  2. Catodo (perna curta do LED) conectado a um resistor de 220Ω, e a outra ponta do resistor conectada ao GND do Arduino.

4. Código do Arduino

O código foi implementado na IDE do Arduino (dentro do Tinkercad) e é responsável por controlar o sensor ultrassônico e acionar o LED com base na distância detectada. O código a seguir foi utilizado:

const int trigPin = 9; const int echoPin = 10; const int ledPin = 13; void setup() { pinMode(trigPin, OUTPUT); pinMode(echoPin, INPUT); pinMode(ledPin, OUTPUT); Serial.begin(9600); } void loop() { long duration; int distance; digitalWrite(trigPin, LOW); delayMicroseconds(2); digitalWrite(trigPin, HIGH); delayMicroseconds(10); digitalWrite(trigPin, LOW); duration = pulseIn(echoPin, HIGH); distance = duration * 0.034 / 2; Serial.print("Distancia: "); Serial.print(distance); Serial.println(" cm"); if (distance <= 10) { digitalWrite(ledPin, HIGH); // Acender o LED } else { digitalWrite(ledPin, LOW); // Apagar o LED } delay(100); }

5. Teste no Tinkercad

O circuito foi simulado no ambiente Tinkercad:

  1. Montagem Virtual: O sensor HC-SR04 foi conectado ao Arduino conforme o esquema elétrico, e o LED foi posicionado corretamente com o resistor em série.
  2. Simulação do Código: O código foi carregado no Arduino virtual do Tinkercad, e a simulação foi iniciada.
  3. Teste da Proximidade: A régua do sensor ultrassônico no Tinkercad foi utilizada para ajustar a distância de um objeto. Quando o objeto estava a 10 cm ou menos, o LED acendia. Ao afastar o objeto para uma distância maior, o LED apagava.

6. Resultados

Os resultados obtidos durante a simulação foram satisfatórios. O sistema respondeu corretamente à proximidade de um objeto:

  • Quando o objeto estava a 10 cm ou menos do sensor, o LED acendia.
  • Quando o objeto estava a uma distância maior que 10 cm, o LED apagava.

O funcionamento do sensor ultrassônico e o controle do LED pelo Arduino seguiram o comportamento esperado, comprovando a eficácia do sistema de detecção de proximidade.

7. Conclusão

O projeto alcançou seu objetivo de implementar um sistema simples de detecção de proximidade usando um sensor ultrassônico HC-SR04 e um LED controlado por um Arduino. A simulação no Tinkercad demonstrou que o sistema é capaz de acender e apagar o LED de acordo com a distância do objeto detectado, permitindo a visualização em tempo real. Este projeto pode ser ampliado para sistemas mais complexos de detecção de obstáculos, automação de iluminação, ou mesmo controle de robôs. O uso do Tinkercad para simulação também mostrou-se uma ferramenta eficaz para a prototipagem virtual antes da implementação prática.
